The timeline

From one date to a stronger checkout promise.

The work did not replace the first estimate. It strengthened it, proved the change, and moved the result closer to the buyer.

01 Foundation

EDD V1 created the delivery promise.

It turned transit time into a date customers could understand. That foundation gave us something real to measure and improve.

Customer signal 1 date

A simple promise, designed for a complex delivery network.

02 Review

One date could not carry all the uncertainty.

Delivery varies by postcode, carrier, pickup timing, service and calendar. In the reviewed sample, the original date matched the delivered day 40.75% of the time.

03 Recalculation

EDD V1's promise became more accurate.

After the bug fix and after-pickup recalculation, exact-day match rose to 81.63% in the same assessment.

After recalculation 81.63%

Twice the original exact-day match rate in the reviewed set.

04 EDD V2

Confidence now travels with the estimate.

EDD V2 passes confidence levels to merchants, giving them more transparency when choosing which delivery promise to show. Percentiles, actual dates and fallbacks make that promise more usable.

Shape P30, P50 and P75 percentiles
Display Actual dates and date ranges
Trust Confidence passed to merchants
Coverage Fallbacks prevent null responses
05 Display logic

The promise adapts to confidence.

EDD V2 returns the most precise promise the available data can support. High confidence shows one date. Medium confidence shows a date range. Low confidence returns a configured fallback range.

High
Exact date Wed 20 Apr
Medium
Date range Wed 20 - Thu 21 Apr

06 V3 Quote API

EDD now powers V3 Quote API.

V3 Quote API now returns EDD with each quote. The new object is additive, so existing integrations keep their transit-time string while newer surfaces use the richer promise. 

"edd": {
  "earliest": "2026-04-20",
  "latest": "2026-04-21",
  "confidence": "high"
}

If EDD is unavailable, the quote still returns the legacy estimate.

07 Shopify checkout

EDD now powers Shopify checkout.

Shopify Connect requests rates through V3 Quote API. The response now carries EDD into checkout, so Shopify merchants can show a delivery range at the moment a buyer chooses shipping.

Shopify checkout-> Shopify Connect-> V3 Quote API-> EDD
